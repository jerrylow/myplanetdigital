UX Review: Drupal's Panels & Page Manager Modules

  • technology

Recent discussions about architectural shifts in the Drupal core have cast a strong light on the Panels and Page Manager modules which are seen as a model for the way forward. All the more reason to take a good look at the usability issues and ways to improve their overall UX...

Over the last several months Myplanet has been carrying out an extensive UX review of Panels and Page Manager which are two important contributed modules in the Drupal landscape. Interestingly, as we carried out this review the Drupal community has been engaged in some important discussions about how to use Panels and Page Manager as a model for a new paradigm of site building and content management in Drupal’s core. We’re offering this UX review as a contribution to that discussion.

Our Goals

The aim of this UX review is to notate key usability issues that may hamper users of the Page Manager system and various Panels components. The issues are placed in the context of how they might affect two distinct user groups: site builders and content managers.

The material for this document is derived from my expertise as a UX designer/developer and not through usability tests. I gave consideration to both the ‘subjective’ factors that I experienced while using the system as well as the ‘objective’ conditions determined by an examination of the code itself.

My intent was to quickly identify key issues, some of which might be the focus of subsequent usability tests and some of which might be relevant to discussion about ‘Core Context UX’ for Drupal.

Criteria

I examined the system from the point of view of these key dimensions of usability:

  • General identification and interrelation of UI components as they relate to underlying data models and functional objects
  • Layout structure and visual presentation
  • Navigational topology
  • Interaction strategy
  • In-line textual help such as prompts and labels
  • Terminology used
  • Appropriateness of tasks for each target user group in key usage contexts
  • General workflow
  • Alternative Designs

I’ve also included a section at the end of the document to present some possible alternative UX strategies for Page Manager, Panel Page Management and In Place Editing. These suggestions may or may not be directly useful in the context of ‘fixing’ the usability obstacles of these modules. But I do believe that the underlying principles behind these alternative designs will be relevant to envisioning a UX strategy for Drupal’s new architectural paradigm.
